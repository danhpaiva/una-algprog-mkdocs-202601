# 🚀 Missão: Sua Identidade Digital e Primeiro Repositório

Agora que você já entendeu que programar é sobre **decompor problemas** e criar **algoritmos**, chegou a hora de preparar o seu kit de ferramentas de desenvolvedor.

Nesta tarefa, vamos utilizar o **GitHub**, que funciona como o "LinkedIn dos programadores" e nosso principal portfólio de código.

---

## 🛠️ O Algoritmo da Entrega

Para concluir esta atividade, siga este passo a passo (o seu algoritmo de sucesso):

### 1. Criar sua Base de Operações (GitHub)

Se você ainda não tem, crie uma conta no [GitHub](https://github.com/).

* **Dica:** Use um nome de usuário profissional, pois você o utilizará durante todo o curso e possivelmente na sua carreira.

### 2. Networking de Código

Um bom desenvolvedor nunca está sozinho. Procure e siga seus colegas de turma no GitHub.

* Isso ajuda a criar uma comunidade de aprendizado onde vocês podem ver o progresso uns dos outros.

### 3. Criar o Repositório da Disciplina

Crie um novo repositório público com o nome exato:
`una-algprog-lista01`

> **Por que esse nome?** A padronização facilita a organização dos seus projetos e a correção pelo professor.

### 4. Subir seu Algoritmo (Upload)

Lembra do desafio do **"Mestre Cuca"** ou da **"Travessia do Rio"** que fizemos em aula?

* Crie um arquivo de texto (`.txt` ou `.md`) com a solução do algoritmo que você desenvolveu.
* Faça o upload (ou o *commit*) desse arquivo para o seu novo repositório no GitHub.

### 5. Entrega Final no Google Classroom

Com o repositório pronto e o arquivo lá dentro:

1. Copie a **URL do seu repositório** (ex: `https://github.com/seu-usuario/una-algprog-lista01`).
2. Vá até a tarefa correspondente no **Google Classroom**.
3. Cole o link e marque a tarefa como entregue.

---

## ⚠️ Checklist de Sucesso

!!! check "Verifique antes de enviar"
- [ ] O repositório está configurado como **Público**? (Se for privado, eu não consigo ver!).
- [ ] O nome do repositório está correto?
- [ ] O arquivo com o algoritmo está legível dentro do GitHub?
